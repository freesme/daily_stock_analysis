# WebUI 功能测试指南

本文档描述如何测试 WebUI 相关的变更，包括 API 接口、页面功能和实时进度展示。

## 1. 启动 WebUI 服务

```bash
# 方式一：仅启动 WebUI（推荐测试用）
python main.py --webui-only

# 方式二：启动 WebUI + 执行一次分析
python main.py --webui
```

启动成功后会显示：
```
WebUI 运行中: http://127.0.0.1:8000
通过 /analysis?code=xxx 接口手动触发分析
```

## 2. 测试 API 接口

### 2.1 健康检查

```bash
curl http://127.0.0.1:8000/health
```

**预期返回：**
```json
{"status": "ok", "timestamp": "2026-01-19T10:30:00.000000"}
```

### 2.2 触发单只股票分析

```bash
curl "http://127.0.0.1:8000/analysis?code=600519"
```

**预期返回：**
```json
{
  "success": true,
  "task_id": "abc123-def456-...",
  "message": "分析任务已提交",
  "code": "600519"
}
```

### 2.3 查询所有任务状态

```bash
curl http://127.0.0.1:8000/tasks
```

**预期返回：**
```json
{
  "success": true,
  "tasks": [
    {
      "task_id": "abc123-def456-...",
      "code": "600519",
      "status": "running",
      "start_time": "2026-01-19T10:30:00.000000"
    }
  ],
  "total": 1
}
```

### 2.4 查询单个任务状态

```bash
curl "http://127.0.0.1:8000/task?id=<task_id>"
```

**运行中返回：**
```json
{
  "success": true,
  "task": {
    "task_id": "abc123-def456-...",
    "code": "600519",
    "status": "running",
    "start_time": "2026-01-19T10:30:00.000000"
  }
}
```

**完成后返回：**
```json
{
  "success": true,
  "task": {
    "task_id": "abc123-def456-...",
    "code": "600519",
    "status": "completed",
    "start_time": "2026-01-19T10:30:00.000000",
    "end_time": "2026-01-19T10:31:30.000000",
    "result": {
      "name": "贵州茅台",
      "operation_advice": "观望",
      "sentiment_score": 75,
      "trend_prediction": "震荡",
      "analysis_summary": "..."
    }
  }
}
```

## 3. 测试页面功能

### 3.1 访问配置页面

1. 打开浏览器访问 `http://127.0.0.1:8000`
2. 确认页面正常显示，包含两个区域：
   - **配置管理** - 自选股列表编辑
   - **快速分析** - 单只股票分析触发

### 3.2 测试快速分析组件

| 操作 | 预期结果 |
|------|----------|
| 输入少于6位数字 | 按钮保持禁用状态（灰色） |
| 输入非数字字符 | 输入被阻止 |
| 输入6位股票代码（如 `600519`） | 按钮变为可用（蓝色） |
| 点击「开始分析」 | 任务卡片出现，显示「分析中...」 |
| 按 Enter 键 | 等同于点击按钮 |

### 3.3 测试实时进度

1. **提交任务后**
   - 任务卡片立即出现
   - 显示股票代码和「分析中...」状态
   - 显示旋转的加载图标

2. **分析过程中**
   - 任务状态每 3 秒自动刷新
   - 耗时持续更新
   - 点击任务卡片头部可展开/收起详情

3. **分析完成后**
   - 状态变为 ✅ 已完成
   - 详情区显示分析结果：
     - 股票名称
     - 操作建议（高亮显示）
     - 综合评分
     - 趋势预测
     - 分析摘要
   - 显示「分析完成，通知已推送」提示
   - 出现删除按钮（🗑️）

4. **分析失败时**
   - 状态变为 ❌ 分析失败
   - 详情区显示错误信息
   - 出现删除按钮

### 3.4 测试多任务并行

1. 连续提交多个股票代码：
   - `600519`（贵州茅台）
   - `300750`（宁德时代）
   - `002594`（比亚迪）

2. 验证：
   - 多个任务卡片同时显示
   - 运行中的任务排在最前面
   - 各任务进度独立更新
   - 完成顺序可能与提交顺序不同

## 4. 测试边界情况

| 测试场景 | 操作 | 预期结果 |
|----------|------|----------|
| 无效股票代码 | 输入 `999999` 并提交 | 分析失败，任务卡片显示错误信息 |
| 空代码提交 | `curl "/analysis?code="` | 返回 `{"success": false, "error": "..."}` |
| 缺少参数 | `curl "/analysis"` | 返回 `{"success": false, "error": "缺少 code 参数"}` |
| 重复提交 | 同一代码连续提交多次 | 创建多个独立任务，各自运行 |
| 查询不存在的任务 | `curl "/task?id=invalid"` | 返回 `{"success": false, "error": "任务不存在"}` |
| 服务重启 | 重启后访问 `/tasks` | 任务列表清空（内存存储，不持久化） |
| 轮询超时 | 等待超过 6 分钟 | 任务卡片显示「轮询超时」错误 |

## 5. 验证通知流程

分析完成后，检查配置的通知渠道是否收到推送：

- [ ] 企业微信
- [ ] 飞书
- [ ] Telegram
- [ ] 邮件
- [ ] 自定义 Webhook

**通知内容应包含：**
- 股票代码和名称
- 操作建议
- 买入/止损/目标价位
- 分析摘要

## 6. 代码结构验证

确认以下文件存在且功能正常：

```
web/
├── __init__.py      # 模块导出
├── server.py        # HTTP 服务器
├── router.py        # 路由管理
├── handlers.py      # 请求处理器
├── services.py      # 业务服务（分析任务管理）
└── templates.py     # HTML 模板（含 JS 逻辑）
```

## 7. 常见问题排查

### Q: 页面无法访问

```bash
# 检查端口是否被占用
netstat -ano | findstr :8000

# 尝试指定其他端口
set WEBUI_PORT=8080
python main.py --webui-only
```

### Q: 分析任务一直显示「分析中」

1. 检查日志文件 `logs/stock_analysis_*.log`
2. 确认 AI API Key 配置正确
3. 确认网络连接正常

### Q: 通知未收到

1. 检查 `.env` 中通知渠道配置
2. 查看日志中的通知发送记录
3. 确认 Webhook URL 有效
